# Grammar

## Overview

This guideline outlines essential grammar and writing style rules for all product content, ensuring consistency, clarity, and a professional voice.

## Capitalization

### Sentence Case
Always use sentence case for headings, labels, and general text. Only capitalize:
- First word of a sentence/phrase
- Proper nouns

| Do | Don't |
|:---|:------|
| Create proposal | Create Proposal |
| Unable to send proposal | Unable To Send Proposal |
| 50 clients selected | 50 Clients selected |

### Proper Nouns and Acronyms

| Do | Don't |
|:---|:------|
| John Smith | john smith |
| Bacs | BACS |
| GST | gst |
| Services library | Services Library |
| QuickBooks | Quickbooks |

| Center | Centre |

## Date Formats

| Format | AU/UK | US |
|:-------|:------|:---|
| `longMonth` | 28 February 2019 | February 28, 2019 |
| `shortMonth` | 28 Feb 2019 | Feb 28, 2019 |
| `longWithTime` | 28 February 2019 8:00 AM | February 28, 2019 8:00 AM |
| `numeric` | 28/2/2019 | 2/28/2019 |
| `isoDate` | 2019-02-28 | 2019-02-28 |

## Time Formats (Relative)

| Time Since Event | Format |
|:-----------------|:-------|
| Less than 1 minute | a few seconds ago |
| Less than 2 minutes | a minute ago |
| Less than 60 minutes | 10 minutes ago |
| Less than 2 hours | an hour ago |
| Less than 24 hours | 10 hours ago |
| More than 24 hours | 16 January 2022 11:45 AM |

## Dashes and Hyphens

| Type | Use For | Example |
|:-----|:--------|:--------|
| **Hyphen (-)** | Joining words, prefixes, blank table fields | client-specific, pre-existing |
| **En dash (–)** | Ranges, numbers | 2023–2024, Monday–Friday |
| **Em dash (—)** | Adding extra information (use sparingly) | "Your demo proposal—that's us." |

## Personal Pronouns

| Pronoun | When to Use | Example |
|:--------|:------------|:--------|
| You, your | Questions, instructions, descriptions | "You'll get an email when your client accepts" |
| I, my | User making a choice/performing action | "I accept the terms and conditions" |
| We, us | Ignition taking responsibility/acting for user | "There was a problem on our end" |

## Bold Text

Use bold for:
- **Key numbers/data** - Transaction numbers, card numbers, dollar values
- **UI element names** in product tours
- **Terms requiring emphasis** in bulleted lists

## Bulleted Lists

- **Do:** Use end punctuation for complete sentences
- **Don't:** Add end punctuation for single words
- **Don't:** Add end punctuation if the item is a link

## Numbers

- Use commas for numbers with 4+ digits: `4,000`
- Currency after dollar amount: `$400 USD`
- Truncation: `$5K` (no space between number and K/M)
- If $0, display as `$0` or `$0.00` (matching decimal precision)
