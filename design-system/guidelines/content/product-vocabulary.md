# Product Vocabulary

## Overview

Standard product terminology to be used across all product interfaces and documentation. Consistent language ensures clarity and reduces ambiguity.

## Action Language

### Add Actions

| Term | Meaning | Example |
|:-----|:--------|:--------|
| **Add** | Add something existing to a container | Add service (to a proposal) |
| **Create** | Create something from scratch | Create proposal, Create template |
| **Duplicate** | Create a copy of existing | Duplicate proposal |
| **New** (adjective) | Use on index pages, follow with noun, include `+` | `+ New proposal`, `+ New client` |

### Modify Actions

| Term | Meaning | Example |
|:-----|:--------|:--------|
| **Change** | Change from one thing to another | Change payment method |
| **Edit** | Modify an existing item | Edit service |
| **Mark as...** | Change status | Mark as lost |
| **Save** | Keep changes made | Save changes (to settings) |

### Delete Actions

| Term | Meaning | Example |
|:-----|:--------|:--------|
| **Archive** | Move to inactive state (still exists) | Archive client |
| **Cancel** | Don't continue | Cancel subscription |
| **Delete** | Permanently erase (no longer accessible) | Delete proposal |
| **Discard** | Exit without saving | Discard changes |
| **Revoke** | Cancel a proposal/contract | Revoke proposal |
| **Remove** | Erase from container, not database | Remove service (from proposal) |

### Other Actions

| Term | Meaning | Example |
|:-----|:--------|:--------|
| **Import** | One-off data transfer, user-initiated | Import clients |
| **Sync** | Regular, often automatic data synchronization | Sync clients |
| **Export** | Extract to different file format | Export CSV |
| **Download** | Save file from web to computer | Download PDF |
| **View** | Navigate somewhere (no action required) | View billing schedule |
| **Schedule** | Set action for future time | Schedule payment collection |

## Payments Terminology

### Collections and Payouts

| Term | Part of Speech | Meaning |
|:-----|:---------------|:--------|
| **Collection** | Noun | Payment taken from client's bank account |
| **Collecting** | Verb present | Payment being taken |
| **Collected** | Verb past | Payment successfully taken |
| **Payouts** | Noun | Money paid to customer after collecting |
| **Paying out** | Verb present | Money being sent |
| **Paid out** | Verb past | Successfully sent to customer's bank |

### Payment Details Format

| Payment Method | Format | Example |
|:---------------|:-------|:--------|
| Credit/debit card | `[CARD PROVIDER] •••• [Last 4]` | VISA •••• 4242 |
| Direct debit/ACH/PAD | `[BANK NAME] •••• [Last 4]` | UNITED BANK •••• 5004 |

## People Terminology

### Client vs. Customer

- **Client** = The customer's clients (end users of proposals/services)
- **Customer** = Users of Ignition (businesses using the platform)

| Do | Don't |
|:---|:------|
| "You're about to cancel this payment collection." | "The customer is about to cancel..." |
| "Ignition customers save..." | "Ignition clients save..." |

## Billing vs. Invoicing

- **Invoicing** = Task of creating and sending invoices
- **Billing** = Broader process of managing money owed
- **Invoice (noun)** = Document sent to client to request payment
- **Bill (noun)** = Document showing what customer owes

| Do | Don't |
|:---|:------|
| Use "invoice" for money owed TO customer | Use "bill" and "invoice" interchangeably |
| Use specific actions: "Create invoice", "Send invoice" | Use "Bill client" or "Invoice now" |
| Use "Billing" for pre-invoice setup | - |

## Renewal Terminology

| Term | Type | Definition |
|:-----|:-----|:-----------|
| **Renew** | Verb, present | Create a proposal with renewal services |
| **Renewed** | Verb, past | A proposal with renewal services was created |
| **Renewal** | Noun | A proposal containing renewal services |
| **Renewal service** | Adjective + noun | Service continuing beyond original proposal |
