# Icons

## Overview

Icons are fundamental visual elements representing actions, features, statuses, and navigation. This guideline provides principles for consistent icon selection and appropriate usage.

## Icon Weight

| Weight | Description | Usage |
|:-------|:------------|:------|
| **Regular** | Default icon weight | Most contexts |
| **Light** | Thinner strokes | Menu items, icons > 14pt |
| **Solid** | Filled, prominent | Toasts on white backgrounds |

## Icon Selection by Context

### Content Management

| Concept | Icon | Notes |
|:--------|:-----|:------|
| Add | `plus` | Add to a set/grouping |
| Edit | `pencil` | Modify existing item |
| Remove | `minus` | Remove from set (not delete) |
| Delete | `trash` | Permanently delete |
| Approve | `square-check` | Approve an item |
| Accept/Positive | `check` | Set positive state |
| Cancel/End | `circle-xmark` | Cancel process |
| Revoke/Undo | `rotate-left` | Undo an action |
| Schedule | `calendar-lines-pen` | Set time for action |
| Re-do | `rotate-right` | Refresh, retry |
| Move | `arrow-right-long` | Move items/statuses |

## Best Practices

### Do's
- Use widely recognizable icons based on common patterns
- Maintain consistency - same icon for same concept
- Always add tooltips to icon-only buttons

### Don'ts
- Don't use multiple different icons for the same concept
- Don't use visually unbalanced icons
- Don't use obscure or unfamiliar icons
